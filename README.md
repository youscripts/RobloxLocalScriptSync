# RobloxLocalScriptSync

**Синхронизация Lua-скриптов между игроками в Roblox через WebSocket.**

Этот проект позволяет тебе и твоим друзьям подключаться к общей сессии и выполнять Lua-скрипты одновременно. Идеально подходит для тестирования скриптов, создания эффектов или просто для веселья в компании.

---

## 🚀 Как подключиться к друзьям (инструкция для всех)

Чтобы начать, всем участникам нужно выполнить всего **один** шаг:

1.  **Скопируйте и вставьте эту команду в ваш Executor (Delta, KRNL, Synapse и т.д.):**

```lua
loadstring(game:HttpGet("https://raw.githubusercontent.com/youscripts/RobloxLocalScriptSync/refs/heads/main/Lua%20Client%20Session%20Script"))()
Примечание: Убедитесь, что у вас установлен и запущен Executor, который поддерживает WebSocket-соединения.

После выполнения скрипта на экране появится небольшое окно для входа в сессию.

🎮 Как это работает
Главное окно: Все видят маленькое окошко с полем для ввода 5-значного кода и двумя кнопками: Войти и Создать.

Создание сессии: Если ты нажмешь Создать, сервер сгенерирует уникальный 5-значный код. Ты станешь создателем сессии.

Подключение: Твои друзья вводят этот код в свое окно и нажимают Войти. Теперь вы в одной сессии!

Выполнение скриптов:

Создатель: Видит поле для ввода скрипта и кнопку Выполнить. Любой скрипт, отправленный создателем, будет выполнен у всех участников сессии одновременно.

Участники: Видят только окно вывода, куда будут приходить результаты выполнения скриптов (например, сообщения от print()).

✨ Все фичи
Простое подключение: Присоединяйся к друзьям по уникальному 5-значному коду.

Синхронное выполнение: Запускай скрипты, которые выполняются у всех в сессии одновременно.

Общий вывод: Все результаты выполнения скриптов (например, print()) видны в окне каждого участника.

Два режима:

Создатель: Может отправлять скрипты и управлять сессией.

Участник: Подключается и видит результаты.

Удобный интерфейс:

Маленькое и простое окно, которое можно перетаскивать.

Кнопка — сворачивает окно, а зеленая кнопка + в углу экрана позволяет его восстановить.

Автоматический выход: Если игрок закрывает окно сессии или его персонаж перерождается (respawn), он автоматически покидает сессию.

Закрытие сессии: Создатель может закрыть сессию, и всех участников выкинет из нее.

Кроссплатформенность: Работает на любом устройстве, где есть Roblox Executor с поддержкой WebSocket.

🧠 Как устроена сессия
Сервер (Node.js + WebSocket) обрабатывает все подключения и хранит активные сессии.

Каждая сессия идентифицируется 5-значным кодом (латиница + цифры), который генерируется автоматически при создании.

Создатель сессии – тот, кто её создал. Только он может:

Отправлять скрипты на выполнение.

Закрыть сессию (принудительно отключить всех).

Участники подключаются по коду, видят вывод скриптов и могут только получать выполняемые скрипты.

Все сообщения между клиентом и сервером передаются в формате JSON (текстовый протокол).

Сессия автоматически удаляется с сервера, когда из неё выходит последний участник.

При перезаходе в игру или респавне игрок автоматически покидает сессию (чтобы избежать «висячих» подключений).

⚠️ Лимиты и ограничения
Параметр	Значение
Максимум участников в сессии	Не ограничено технически, но рекомендуется не более 10–15 для стабильной работы WebSocket.
Длина кода сессии	5 символов (A-Z, 0-9)
Максимальный размер скрипта	Зависит от лимитов WebSocket (обычно до 1 МБ), но лучше не превышать 10–20 КБ, чтобы не тормозить передачу.
Время жизни сессии	Сессия существует, пока в ней есть хотя бы один участник. При выходе последнего сессия удаляется.
Повторное подключение	Если вы вышли из сессии, можете войти снова по тому же коду (если сессия ещё активна).
Одновременные сессии	Сервер поддерживает неограниченное количество сессий одновременно.
Скорость передачи	Зависит от вашего интернет-соединения и загруженности сервера Render (бесплатный тариф может замедляться при большой нагрузке).
